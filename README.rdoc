Используя Rails, нужно разработать приложение для сбора и базового анализа логов доступа к веб-серверам (нечто похожее на крайне упрощенную версию Google/Яндекс анатилики). Это приложение должно состоять из двух частей:
 
 
1. Веб-интерфейс.
 
Будет отвечать за подключение к приложению лог-файлов и отображение полученной из них информации. Подключение лога к системе должно происходить посредством ввода URL-адреса, по которому распологается лог-файл (например "http://www.almhuette-raith.at/apache-log/access.log"). Можно считать, все добавляемые в систему файлы будут соответствовать стандартному формату Апачевских access.log'ов, описанному по ссылке [1]. Дополнительные примеры лог-файлов в этом формате можно искать по другой ссылке [2].
 
Для всех лог-файлов должна присутствовать возможность вывода в табличном виде полученной из них информации о HTTP-запросах. Эта таблица должна содержать следующие столбцы: время запроса, код ответа, длинну ответа, HTTP-метод запроса, адрес запроса и User-Agent посетителя. Для данной таблицы надо реализовать:
  * Возможность сортировки строк по неубыванию/невозрастанию значений стоблцов.
  * Возможность задавать для каждой колонки регулярное выражение, фильтрующее записи в таблице по соответствующему полю.
 
 
2. Бэкенд.
 
Эта часть будет отвечать за скачивание и анализ лог-файлов. Она должна проводить как первоначальную загрузку логов (при добавлении нового URL'а в систему), так из периодическое обновление информации о новых запросах из ранее добавленных URL'ов. Крайне желательно, чтобы процедура обновления лог-файла не качала каждый раз весь файл целиком, а брала только дельту (информацию о новых HTTP-запросах). Настраивать в приложении cron или очередь задач не обязательно, главное, чтобы в коде были четко прописаны процедуры первоночальной загрузки, обновления и анализа логов.
 
 
Главными критериями оценки Вашего решения будет простота и разумность кода, а так же эргономичность реализованного Вами веб-интерфейса проекта.
